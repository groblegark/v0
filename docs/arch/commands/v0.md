# v0

**Purpose:** Main CLI entry point and command dispatcher.

## Workflow

1. Parse command from arguments
2. Handle special commands (`init`, `help`, `version`)
3. Check command type and dispatch to `v0-<command>`

## Usage

```bash
v0 init                              # Initialize project (user-specific branch + local agent remote)
v0 init --develop v0/develop         # Use shared v0/develop branch instead
v0 init --remote origin              # Push to shared origin instead of local agent
v0 build auth "Add JWT auth"         # Full autonomous pipeline
v0 fix "Button not working"          # Report a bug
v0 status                            # List all operations
```

## Init Defaults

| Setting | Default | Description |
|---------|---------|-------------|
| `V0_DEVELOP_BRANCH` | `v0/agent/{username}-{id}` | Unique per-user branch (stored in `.v0.profile.rc`) |
| `V0_GIT_REMOTE` | `agent` | Local bare repo at `~/.local/state/v0/${PROJECT}/remotes/agent.git` |

When no `--develop` flag is provided, `v0 init` creates a `.v0.profile.rc` file with the auto-generated branch name. This file is gitignored so each developer gets their own branch.
